# Как редактировать сайт (без кода)

Весь контент сайта — тексты, даты, ссылки на песни, причины, подписи к
полароидам — лежит в **одном файле**: `src/content.ts`. Больше никуда лезть
не нужно.

## 1. Как открыть файл на GitHub

1. Зайди в репозиторий на github.com.
2. Открой `src/content.ts`.
3. Нажми на иконку карандаша (Edit this file) в правом верхнем углу файла.
4. Правь текст прямо в браузере.
5. Внизу страницы нажми **Commit changes** (можно сразу в `main`).

Готово — через 1–2 минуты сайт пересоберётся и обновится сам (см. пункт 5).

## 2. Как добавить фото

1. В репозитории зайди в папку `public/img/`.
2. Нажми **Add file → Upload files**, загрузи картинку (например `first-call.jpg`).
3. Commit changes.
4. Открой `src/content.ts`, найди нужное место и впиши путь вида `"/img/first-call.jpg"`.

В таймлайне (`story`) у каждого события есть поле `images` — это список
фотографий, можно 0, 1, 2 или больше, они сами красиво лягут рядом.

Пример — было:
```ts
{ date: "2025-08-02", title: "First call", text: "...", images: [] }
```
стало (одна фотография):
```ts
{ date: "2025-08-02", title: "First call", text: "...", images: ["/img/first-call.jpg"] }
```
или сразу две:
```ts
{ date: "2025-08-02", title: "First call", text: "...", images: ["/img/first-call.jpg", "/img/first-call-2.jpg"] }
```

`date` можно вообще не указывать, если не помнишь точную дату — просто убери
строку `date: "..."` целиком, ничего не сломается. На сайте фото в таймлайне
теперь кликабельны — открываются на весь экран, удобно смотреть, если фото
несколько и они мелкие.

## 3. Как «оживить» полароид Future Memories

Найди в `content.ts` блок `futureMemories`. У каждой карточки есть `caption`
(подпись) и `image` (сейчас `null`). Как только событие случилось и есть фото —
загрузи фото в `public/img/` (см. пункт 2) и замени `null` на путь к файлу.
Рамка сама «оживёт» на сайте — картинку добавлять больше нигде не нужно.

## 4. Как добавить причину / событие / трек

**Причина** (раздел Reasons) — просто добавь новую строку в массив `reasons`:
```ts
reasons: [
  "...",
  "Because you always call when you say you will.", // ← новая строка
],
```

**Событие таймлайна** (Our Story) — добавь объект в массив `story`:
```ts
{ date: "2026-01-05", title: "Заголовок", text: "Текст истории", images: [] },
```

**Трек** (Our Music → плейлист) — добавь объект в `music.playlist.tracks`:
```ts
{ title: "Название трека", artist: "Исполнитель", youtubeId: "ID_ИЗ_YOUTUBE" },
```
`youtubeId` — это часть ссылки после `v=`. Например, для
`youtube.com/watch?v=ABC123xyz` id — `ABC123xyz`.

Для главного трека «La Vie en Rose» тот же принцип — впиши `youtubeId` в
`music.featured`.

## 5. Как добавить послание «A message from me»

По умолчанию `message: null` — раздел и пункт меню скрыты полностью, ничего
не сломается. Чтобы раздел появился, замени `null` на объект с тремя
вариантами `type`:

**Видео с YouTube (рекомендую)** — загрузи видео на YouTube как **Unlisted**
(«Доступ по ссылке», не публичное), скопируй ID из ссылки после `v=`:
```ts
message: { type: "youtube", src: "ABC123xyz", note: "Необязательная подпись" },
```

**Свой видеофайл** — загрузи `.mp4` в папку `public/video/` (создай её так же,
как `public/img/`, через Add file → Upload files) и укажи путь:
```ts
message: { type: "video", src: "/video/message.mp4" },
```
Учти: через веб-интерфейс GitHub нельзя загрузить файл тяжелее ~25 МБ —
для видео поэтому обычно проще вариант с YouTube выше.

**Свой аудиофайл** — так же, но в `public/audio/`:
```ts
message: { type: "audio", src: "/audio/message.mp3" },
```

`note` у всех вариантов необязательный — короткая подпись под плеером.

## 6. Что происходит после сохранения

После **Commit changes** в GitHub автоматически запускается сборка (GitHub
Actions) — она собирает сайт и выкладывает его на сервер. Обычно занимает
1–2 минуты. Прогресс можно посмотреть во вкладке **Actions** репозитория:
зелёная галочка — всё выложилось, красный крестик — что-то не собралось
(обычно опечатка в `content.ts`, например забытая запятая или кавычка).

Если сборка упала — самое частое: пропущенная запятая между объектами в
массиве или незакрытая кавычка `"`. Открой файл заново, сравни со
структурой примеров выше.

## 7. Раскрытие в годовщину (опционально)

В самом верху `content.ts` есть блок:
```ts
reveal: { enabled: false, at: "2026-07-17T00:00:00+03:00" }
```
По умолчанию выключено (`enabled: false`) — сайт сразу открывается конвертом.
Если хочешь, чтобы до годовщины показывался обратный отсчёт, а сайт
раскрывался ровно в момент `at` — поставь `enabled: true`.

## 8. Чего НЕ трогать

Не редактируй файлы вне `src/content.ts` и папки `public/img/`, если не
уверен, что делаешь — это код сайта, ошибка в нём может всё сломать. Если
хочется что-то поменять в дизайне или добавить раздел — лучше попроси
Claude Code сделать это, а сам продолжай редактировать только `content.ts`.
